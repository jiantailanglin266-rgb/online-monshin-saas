# Phase 7：AI追加質問機能 実装メモ（実装前設計＋自己レビュー）

- 作成日：2026-07-03

## 1. 実装スコープ
1. **AiGateway**：プロバイダ抽象化（Anthropic / OpenAI / デモ用モック）＋ ai_logs 記録＋出力バリデーション＋リトライ＋フォールバック（PHASE4 §4）
2. プロンプト実装：①初回問診解析／②追加質問生成／⑥危険症状検知（AI_PROMPTS.md v1 の3本。③④⑤は Phase 8）
3. インタビューAPI：`GET /interview`（状態）／`POST /interview/next`（次の質問）／`POST /interview/answers`（回答保存＋危険判定）
4. S-06 チャットUI（吹き出し・選択式クイックリプライ・「ここまでの内容で終わる」）
5. submit 時に ①初回解析（interview_plan生成）＋⑥危険検知 を実行

## 2. 設計判断
| 論点 | 判断 | 理由 |
|---|---|---|
| プロバイダ呼び出し | SDKを追加せず fetch 直叩き（Anthropic Messages API / OpenAI Chat Completions） | 依存を増やさない。タイムアウトはAbortController |
| デモモード | **MockAiProvider**：interview_plan と質問をカテゴリ別の決定的テンプレートから生成（遅延なし・API키不要） | env無しでもチャットフローを完全に体験できる（本プロジェクトの一貫方針） |
| 質問の重複実行防止 | `interview/next` は「未回答の質問があればそれを返す」。生成は未回答ゼロのときのみ（PHASE4 §6 の対策） | 連打・リロードでAI呼び出しが多重にならない |
| 回答時の危険判定 | ルールエンジンを**全回答で同期実行**（テキスト走査関数を rules.ts から分離）。AI⑥は3問ごと（実プロバイダ時のみ） | デモ時もルール側で危険フローを再現できる |
| 質問終了の判定 | ①8問到達 ②プランの全トピック消化 ③AI が done=true ④患者スキップ、のいずれか | 患者を拘束しない（S-06「ここまでの内容で終わる」常設） |
| interview_plan の保存先 | medical_questionnaires.interview_plan（Json、スキーマ定義済み） | 別テーブル不要 |
| 禁止表現検査 | 患者に露出する question.text のみ対象（AI_PROMPTS.md §7 のパターン）。検知→リトライ1回→フォールバック定型質問 | SAFE-1 二重ガード |

## 3. 自己レビュー（実装前）
| 指摘 | 対応 |
|---|---|
| submit時のAI呼び出し（①+⑥）で送信完了が遅くなる | デモ=即時。実プロバイダ時はsubmitレスポンスを待たせず、①はinterview/next初回時に遅延実行（planが無ければその場で生成）。⑥のみsubmitで実行(安全優先) |
| AIが同じ質問を繰り返す | consumed topics を毎回プロンプトに渡し、生成後に「既出質問と同一文」の重複チェックをアプリ層でも行う |
| 自由入力回答へのプロンプトインジェクション | 回答はJSON文字列としてcontextに埋め込み（<patient_data>タグ内）。共通プリアンブルの耐性指示＋出力はzodで構造検査 |
| モックが本番コードに紛れる危険 | Phase 5と同じ隔離方針：mockは `lib/ai/mock.ts`、gateway が isDemoMode で選択。実プロバイダ設定時はmockに到達不可 |
| AI応答のJSONがコードフェンス付きで返る | パース前に ```json フェンス除去の正規化を挟む |
| 8秒タイムアウト時にチャットが無言になる | gateway がフォールバック定型質問を返す（source='fallback'）。UIは区別なく表示 |
