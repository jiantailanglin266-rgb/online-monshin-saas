# Phase 8：緊急度判定・SOAP・科目提案 実装メモ（実装前設計＋自己レビュー）

- 作成日：2026-07-03

## 1. 実装スコープ
1. AI③緊急度判定／④SOAP要約／⑤診療科目提案（プロンプト実装＋zodスキーマ＋モック）
2. `POST /questionnaires/[id]/finalize`：ai_interview → triaged。③④⑤を実行し、triage_results / ai_summaries / suggestedDepartments を保存
3. **final_level = max(ai_level, rule_level)**（重症側）。ルール判定はヒットなし時 L4 をベースラインとする（ルールは「上げる」専用）
4. S-07 患者向け結果画面（4分類の文言・免責常設・L1は救急ボタン）
5. チャット完了・スキップ時に finalize → 結果画面へ自動遷移。マイページから結果画面への導線

## 2. 設計判断
| 論点 | 判断 | 理由 |
|---|---|---|
| rule_level のベースライン | ヒットなし=L4（最軽）。L1/L2キーワード・バイタルヒット時のみ上がる | ルールをL3ベースにすると max() でL4が構造的に出なくなる。ルールの役割は「見落とし防止の底上げ」であり通常判定はAI側 |
| AI③失敗時 | final = rule_level のみ（ai_level=NULL）。ただし**ルール単独でL4は危険**（AIの視点なしで「経過観察」を出さない）ため、AI無しの場合の下限は L3 | 見落とし最小化（R1）。PHASE4 §4.6 の具体化 |
| 患者向けL1結果 | 結果画面でも 119/#7119 ボタンを最上部に表示（S-07E同等の行動導線） | finalize時に初めてL1になるケース（AI③のみ検知）がある |
| SOAP保存形式 | s_text等は表示用の結合テキスト、source_refs に文単位の構造（{text, refs[]}）を保存 | Phase 9 の原文対照ハイライトに必要 |
| 患者向け表示の範囲 | 結果画面は final_level の定型文言＋科目候補のみ。**AI根拠・SOAP・uncertainは出さない** | PHASE4 §5（薬機法配慮）。判定根拠は医師画面（Phase 9）のみ |
| 冪等性 | finalize済み（triaged以降）なら既存結果で200を返す | PHASE4 §1.5 |
| モック③ | ルール準拠＋簡易ヒューリスティック（38.5℃以上/痛み7以上→L2、1か月以上前+軽症状→L4、他L3） | デモで4分類すべてを再現可能にする |

## 3. 自己レビュー（実装前）
| 指摘 | 対応 |
|---|---|
| チャット「ここまでで終わる」がクライアント状態のみだと finalize されず宙に浮く | スキップ時も finalize を呼ぶ（未回答質問が残っていても確定できる仕様に） |
| finalize の二重実行で triage_results が重複 | status遷移ガード＋triaged時は最新結果を返す冪等設計。連打はUI側もbusyで抑止 |
| AI③が rule より2段階以上軽い場合はAIの見落とし兆候（PHASE4 §4.5-3） | aiReasons に aiRuleGap フラグを保存（医師画面 Phase 9 で警告表示）。患者への表示は常に重症側なので影響なし |
| L4文言が受診抑制にならないか | Phase 2 の決定どおり「まずは様子を見ることも考えられます（ご心配なら受診いただけます）」＋予約導線をL3と同じ大きさで表示 |
| SOAPのA欄に疾患名断定が混入 | プロンプトで「〜の確認」「〜の除外の検討」形式を強制（AI_PROMPTS.md §4）。モックは定型文のみ |
| 妊娠中・高齢は一段上げるルールがAI任せ | ルール側にも追加：妊娠中(pregnant/possible)+腹痛/出血カテゴリ→L2、65歳以上+発熱38℃以上→L2 |
