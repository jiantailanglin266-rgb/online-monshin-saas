## 1. デザインシステム方針

### 1.1 トーン
清潔感・信頼感・安心感。「病院の待合室の静けさ」を目指し、緊急時のみ明確に色で警告する。

### 1.2 カラーパレット
| 用途 | 色 | 備考 |
|---|---|---|
| ベース背景 | `#F7FAFC`（ほぼ白） | 純白より目に優しい |
| サーフェス | `#FFFFFF` | カード |
| プライマリ | `#2B6CB0`（淡めの医療ブルー） | ボタン・リンク |
| プライマリ淡 | `#EBF4FB` | 選択状態・情報帯 |
| テキスト主 | `#1A202C` | コントラスト比 ≥ 7:1 |
| テキスト副 | `#4A5568` | |
| 緊急L1 | `#C53030`（赤） | **L1のみ赤を使う。他画面で赤は使用禁止** |
| 注意L2 | `#DD6B20`（オレンジ） | |
| 通常L3 | `#2B6CB0`（青） | |
| 経過観察L4 | `#718096`（グレー） | 緑は「安全のお墨付き」に見えるため不使用 |
| 成功 | `#2F855A` | 完了トースト等、緊急度以外の文脈のみ |

### 1.3 タイポグラフィ・サイズ（高齢者配慮）
- フォント：Noto Sans JP
- 本文 **17px** / 補足14px / 見出し22〜28px / 患者向けボタンラベル18px
- タップ領域：最小 **48×48px**、選択肢ボタンは幅100%・高さ56px
- 行間1.7、1行の質問文は40文字以内

### 1.4 レイアウト原則
- **モバイルファースト**（375px基準）。医師ダッシュボードのみPC優先（1280px基準、タブレット対応）
- 患者側：1画面 = 1〜2質問。上部に進捗バー（「あと◯問」表記はAI質問数が可変のため使わず、セクション単位の進捗表示）
- 下部固定の「次へ」ボタン（親指到達圏）
- 入力はキーボードより**選択式ボタンを優先**（体温・自由入力のみキーボード）

### 1.5 共通コンポーネント
`Button` / `ChoiceChip`（選択肢） / `StepProgress` / `TriageBadge`（L1〜L4） / `AiBadge`（「AI生成・未確認」） / `DisclaimerBar`（診断ではない旨の常設帯） / `ChatBubble` / `ImageUploader` / `AppointmentCard` / `AuditableTextarea`（医師メモ）

## 5. UI文言ガイドライン（不安を煽らない・断定しない）

### 5.1 原則
1. **診断・断定をしない**：「〜の可能性があります」も患者向けには使わない（可能性提示自体が診断示唆になり得る）。「医師にご相談ください」へ言い換える
2. **不安を煽らない**：危険を伝えるときは「事実＋行動」だけを簡潔に。感嘆符・赤文字の乱用禁止（赤はL1のみ）
3. **安心させすぎない**：「大丈夫です」「問題ありません」「軽症です」は全面禁止
4. **命令しない**：「〜してください」より「〜をご検討ください」「〜をおすすめします」。ただし119等の行動ボタンは動詞直結
5. **平易に**：中学生でも読める語彙。医学用語には振り仮名または言い換え（例：既往歴→「これまでにかかった病気」）

### 5.2 NG→OK対訳表（抜粋）
| 場面 | ❌ NG | ✅ OK |
|---|---|---|
| 緊急度L1 | 「危険な状態です！今すぐ救急車を！」 | 「すぐに医療機関の受診をご検討ください」 |
| 緊急度L4 | 「心配いりません。様子を見ましょう」 | 「まずは様子を見ることも考えられます。ご心配な場合は受診いただけます」 |
| 科目提案 | 「あなたは皮膚炎の疑いがあります」 | 「ご相談先の候補：皮膚科」 |
| AI質問 | 「肺炎の可能性を調べます」 | 「医師に正確に伝えるため、いくつか質問させてください」 |
| 要約画面 | 「AI診断結果」 | 「AIによる問診の整理（医師確認前）」 |
| エラー | 「AIが利用できません」 | 「自動の追加質問は省略されました。医師が診療時におうかがいします」 |
| 完了 | 「診断が完了しました」 | 「問診が完了しました。内容は医師に共有されます」 |

### 5.3 常設文言
- 患者向け全結果画面フッター：「この表示は受診の目安であり、診断ではありません。最終的な判断は医師が行います。」
- AIチャットヘッダー：「AIアシスタントが、医師に伝えるための情報を整理します。」
- 医師画面：AI生成物すべてに「AI生成・未確認」バッジ（確認操作で消灯）

## 7. 自己レビュー（Phase 2時点の問題点と対応）

| 指摘 | 対応 |
|---|---|
| 「あと◯問」進捗表示はAI質問数が可変のため嘘になる | セクション単位の進捗（基本情報 2/7）に変更。AIパートは進捗数値を出さない |
| L1全画面アラートで問診を完全に打ち切ると、軽微な誤検知時に患者が離脱しクリニックにも情報が残らない | 「症状は落ち着いている・続ける」導線を残しつつ、通知・ログは取り消さない設計に |
| 緊急度L4に緑色を使うと「安全のお墨付き」に見える（薬機法・安全性の両リスク） | L4はグレー。緑は緊急度表示に使用禁止とカラー規定に明記 |
| AIチャットの吹き出しが医師との会話に見える | 「AIアシスタント」明記＋医師アイコンと明確に別デザイン。S-06に規定 |
| SOAP要約だけ見ると原文とズレていても気づけない（automation bias） | S-11を要約⇔原文の対照レイアウトにし、文単位の参照ハイライトを仕様化 |
| 同意の一括チェックは要配慮個人情報の同意として弱い | S-02/S-04で項目別チェック（規約・要配慮情報・AI/外国送信）に分離 |
| 予約画面で緊急度と無関係に枠を出すと、L2患者が3日後を予約してしまう | L2のとき本日枠を最上部に固定表示する仕様を追加 |
| ビデオ入室ボタンが常時押せると患者が誤入室する | 開始15分前に活性化する仕様に |
